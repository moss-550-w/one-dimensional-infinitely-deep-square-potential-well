# 一维无限深方势阱 · 思想之旅

> 一个以「一维无限深方势阱」为唯一物理系统的沉浸式 3D 科学教育网站。它不是线性的物理教科书，而是一次**认知发现之旅**——引导用户从「盒中粒子的混沌」出发，亲历困惑、寻语、顿悟、反思，去发现数学语言（傅里叶分析、希尔伯特空间、量子公理）如何被人类建构出来描述量子世界。

## 核心理念

本项目恪守四条不可妥协的设计原则：

1. **概念诚实性** — 可视化只是认知脚手架，永远不替代抽象概念本身。关键处明确标注「这是简化模型 / 类比，边界在哪里」。
2. **动机内生性** — 每个数学工具的出现，都由用户此前亲历的「描述无力」驱动，绝不使用「某某出现了」式的历史宣告。
3. **物理与数学准确性** — 波函数严格归一化，测量是概率性的、不可逆的本征态坍缩，能量本征值公式正确，傅里叶截断如实呈现吉布斯过冲（绝不假装完美逼近）。
4. **资源生命周期纪律** — 任何 3D 资源从诞生即配套 `dispose()`，离开卫星空间即完全释放，杜绝内存泄漏。

## 旅程结构：核心模拟器 + 卫星章节

网站的锚点是一个贯穿全站的**核心势阱模拟器**，它随用户解锁认知模块逐级升级；六个**卫星章节**围绕核心独立加载、独立销毁。

### 核心模拟器四阶段

| 阶段 | 名称 | 解锁条件 | 数学/物理状态 |
|:---:|------|---------|--------------|
| 0 | 经典混沌 | 初始 | 牛顿力学，随机初值，盒壁弹性碰撞 |
| 1 | 模式分解 | 完成第二章 | 波函数视为傅里叶级数 `ψ = Σ cₙψₙ` |
| 2 | 几何投影 | 完成第三章 | 波函数表示为希尔伯特空间中的向量 |
| 3 | 量子公理操作 | 完成第四章 | 归一化、含时演化、概率坍缩、不确定性 |

### 六个卫星章节

| 章节 | 标题 | 核心体验 |
|:---:|------|---------|
| 第一章 | 起点 | 盒中 500 个经典粒子混沌碰撞，点击选项触发粒子逃逸化作文字 |
| 第二章 | 波的语言 | 傅里叶合成器：用谐波叠加方波，亲见吉布斯过冲永不消失（≈1.179）|
| 第三章 | 无限的几何 | 希尔伯特空间三维类比：矢量约束于单位球面（归一化），正交性演示，折叠的高维轴 |
| 第四章 | 量子公理剧场 | 拖拽制备态、概率性测量坍缩（不可逆）、动量表象（FFT）、波包压缩 `Δx·Δp ≈ ħ/2` |
| 第五章 | 思想的长河 | 3D 力导向知识图谱，四类彩色关系；哥德尔与海森堡间**无连线**，点击空白才浮现「理性的边界」反思 |
| 第六章 | 理性的边界 | 回到阶段3模拟器，灰暗禁用按钮「同时知道确切位置和动量」，开放式问题缓入（无句号收尾）|

## 技术栈

| 层级 | 技术 | 说明 |
|------|------|------|
| 前端 | 原生 HTML/CSS/JavaScript | 不用框架，保证对 3D 场景的完全控制 |
| 3D 引擎 | [Three.js](https://threejs.org/) r160 | InstancedMesh、OrbitControls、自定义几何 |
| 数学计算 | 自研 `QuantumMath` + [Math.js](https://mathjs.org/) | 本征态、傅里叶系数、内积、坍缩抽样、数值傅里叶变换 |
| 动画 | [GSAP](https://gsap.com/) | 场景过渡、时间线控制 |
| 样式 | [Tailwind CSS](https://tailwindcss.com/) | 仅用于基础 UI 面板 |
| 构建 | [Vite](https://vitejs.dev/) 5 | 动态 import 代码分割、Web Worker、GLSL |
| 性能 | Web Worker + 运行时帧率降级 | 计算密集任务移出主线程，自动分级 |
| 测试 | [Vitest](https://vitest.dev/) | 数学内核与状态机纯函数单测 |

## 项目架构

```
src/
├── main.js                      # 应用入口：场景管理器、动态章节加载、帧率降级
├── core/                        # 核心：数学内核 + 状态机 + 阶段可视化
│   ├── QuantumMath.js           # 量子力学与傅里叶数学库（纯函数，全单测）
│   ├── Simulator.js             # 核心模拟器状态机（四阶段）
│   ├── StateBus.js              # 全站事件总线 + 共享状态
│   ├── ClassicalField.js        # 阶段0：经典粒子场（InstancedMesh）
│   ├── ModeDecompositionField.js# 阶段1：模式分解
│   ├── HilbertProjection(Field) # 阶段2：希尔伯特投影（单位球面约束）
│   ├── QuantumAxiomField.js     # 阶段3：含时波函数 + 概率密度云 + 坍缩
│   ├── ForceLayout.js           # 第五章力导向布局引擎（纯计算，确定性）
│   ├── computeKernels.js        # 计算密集纯内核（动量谱）
│   ├── compute.worker.js        # Web Worker（Transferable 回传）
│   ├── ComputeClient.js         # Worker/同步双路 RPC（自动降级）
│   └── PerfMonitor.js           # 运行时帧率监测与自动降级
├── satellites/                  # 六个卫星章节（独立加载/销毁）
│   ├── Chapter1 … Chapter6/
└── shared/
    ├── utils/                   # SceneBase（生命周期基类）、WaveCanvas、textSprite
    └── components/              # 玻璃拟态 UI、旋钮等微组件
```

### 设计要点

- **数学先行，视觉派生** — 每个功能先实现 `QuantumMath` / `ForceLayout` / `computeKernels` 中可单测的纯函数，再驱动视觉层。视觉是数学的派生表现，不允许「看起来对但算错」。
- **生命周期收口** — 所有卫星场景继承 `SceneBase`，通过 `track()` / `listen()` 登记资源，`dispose()` 时递归释放 geometry / material / texture 与事件监听。
- **状态总线** — 模拟器阶段、解锁模块、性能分级集中存于 `StateBus`，跨章节同步；离开章节无需保留 3D 资源即可延续旅程进度。
- **计算解耦** — 动量谱等重计算交给 Web Worker，主线程不掉帧；无 Worker / 低端 / 测试环境自动回退同步路径，两路共用同一份纯内核，杜绝逻辑漂移。

## 物理与数学约束（验收红线）

- **能量本征值**：`Eₙ = n²π²ħ² / (2mL²)`（自然单位 ħ=m=L=1）
- **本征函数**：`ψₙ(x) = √(2/L)·sin(nπx/L)`，阱外为 0
- **归一化**：制备态 `Σ|cₙ|² = 1`，含时演化下守恒
- **测量坍缩**：按 `|cₙ|²` 加权随机，一次性、不可逆（唯「重置」可逆）
- **吉布斯现象**：方波正弦级数过冲峰趋近 `≈1.179`，不随项数消失
- **不确定性**：高斯波包 `Δx·Δp ≥ ħ/2`，海森堡下界
- **希尔伯特投影**：永久标注「三维类比」，第 4/5 维以折叠态出现，明示类比边界
