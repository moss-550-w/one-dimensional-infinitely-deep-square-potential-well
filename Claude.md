## 一、项目概述

**项目名称：** 一维无限深方势阱思想之旅  
**版本：** v2.0 重构版  
**类型：** 沉浸式科学教育网站（3D 交互体验）  
**核心目标：** 以“一维无限深方势阱”为唯一物理系统，引导用户发现数学语言（傅里叶分析、希尔伯特空间）如何被建构以描述量子世界，并反思数学与物理的真实关系。

网站不是线性的历史教科书，而是一个**认知发现之旅**。用户从“粒子在盒中的混沌”出发，经历困惑、寻语、顿悟、反思四个阶段。所有数学概念的引入都必须源自用户亲历的认知困境，而不是强行灌输。

## 二、关键设计原则（不可妥协的约束）

1. **概念诚实性**
   - 可视化只是类比，永远不能替代抽象概念本身。
   - 必须在关键点明确告知用户“这是一个简化模型，边界在哪里”。
   - 拒绝强行缝合历史因果链，展示真实的多线并行的思想发展。

2. **动机内生性**
   - 每个数学工具的出现，必须由用户在此前体验到的“描述无力”驱动。
   - 绝不使用“某某出现了”这种历史宣告。

3. **渐进式核心模拟器**
   - 存在一个贯穿全站的核心3D势阱模拟器，它会随用户旅程解锁新的认知模块而升级。
   - 四个阶段：经典混沌 → 模式分解 → 几何投影 → 量子公理操作。

4. **卫星空间架构**
   - 不再使用长滚动单页，而是核心 + 可独立加载和销毁的卫星章节场景。
   - 资源生命周期严格管理，离开即释放。

5. **绝对尊重物理与数学准确性**
   - 波函数必须满足归一化，测量必须是概率性的坍缩到本征态（不可逆），能量本征值公式正确。
   - 傅里叶级数截断时，必须展现吉布斯现象，不能假装完美逼近。
   - 希尔伯特空间的3D投影必须标明“类比”，并明确告知其维度限制。

## 三、目标用户与响应式策略

- **桌面端：** 完整3D交互，自由旋转、拖拽、参数调节。
- **平板端：** 简化3D操作，保留核心交互。
- **移动端：** 自动降级为2.5D预渲染动画 + 热点交互模式，确保核心概念传达，牺牲自由3D旋转。
- **性能最低门槛：** 中端智能手机（如骁龙7系列或等效性能），30fps可接受。

## 四、技术栈

| 层级           | 技术                        | 说明 |
|----------------|-----------------------------|------|
| 前端框架       | 原生 HTML/CSS/JavaScript    | 不使用React/Vue等，保证对3D场景的完全控制 |
| 3D引擎         | Three.js (r150+)            | 包含 OrbitControls、InstancedMesh、ShaderMaterial |
| 数学计算       | Math.js                     | 傅里叶系数计算、本征值、积分、复数运算 |
| 动画库         | GSAP (ScrollTrigger)        | 场景过渡、UI动画、时间线控制 |
| CSS框架        | Tailwind CSS                | 仅用于基础UI面板，不干扰3D画布 |
| 状态管理       | 自定义 PubSub/EventBus      | 核心模拟器状态跨章节同步 |
| 构建工具       | Vite                        | 快速开发，支持 Web Worker 和 GLSL 导入 |
| 性能辅助       | Web Worker                  | 将傅里叶合成、矩阵运算等计算密集型任务移出主线程 |

### 5.2 核心模拟器 (Core Simulator) 状态机

模拟器是整个网站的锚点对象。它的四个阶段由用户在卫星空间中解锁的“认知模块”触发。

| 阶段 | 名称             | 解锁条件             | 可视化表现                               | 数学/物理状态                      |
|------|------------------|----------------------|------------------------------------------|-----------------------------------|
| 0    | 经典混沌         | 初始                 | 大量经典粒子无规则碰撞，无规律          | 随机初始条件，牛顿力学            |
| 1    | 模式分解         | 完成第二章           | 粒子轨迹消失，显示波动曲线+模式分解器   | 波函数视为可分解的傅里叶级数      |
| 2    | 几何投影         | 完成第三章           | 希尔伯特空间投影视图：矢量+坐标轴       | 波函数表示为希尔伯特空间中的向量  |
| 3    | 量子公理操作     | 完成第四章           | 完整量子力学模拟器，可制备态、测量      | 归一化、概率性坍缩、不确定性原理  |

**实现要点：**
- `Simulator` 类持有 `currentStage` 和 `unlockedModules[]`。
- 每次阶段升级触发 GSAP 驱动的转场动画。
- 在阶段1-3中，模拟器内部数学表示必须是精确的（求解薛定谔方程的本征态，或根据系数合成波函数），视觉效果由此派生。

## 六、卫星章节关键实现细节

### 第六章.1 起点（Chapter 1）

- 直接呈现阶段0的核心模拟器。
- 粒子系统使用 `THREE.InstancedMesh` 优化，最多500个粒子。
- 实现简单的经典力学碰撞检测（与盒壁的弹性碰撞）。
- 点击选项触发一个粒子的逃逸动画（轨迹从三维空间“飞”向摄像机，变成文字）。

### 第六章.2 傅里叶合成器（Chapter 2）

- **数学核心：** 实时计算前N项傅里叶级数对目标方波的逼近。
  - 目标函数：`f(x) = 1` 在 `(0, L)`，`0` 在边界。使用正弦级数：`f(x) = Σ (4/(π(2k-1))) sin((2k-1)πx/L)`。
- **交互：**
  - 独立的谐波发生器（最多20个），每个可独立调节振幅和相位（以旋钮形式）。
  - 叠加曲线实时绘制在2D Canvas上（作为纹理贴到3D平面），方便2D精度观测。
  - 吉布斯过冲高亮显示：在合成的方波跳变处，动态标注过冲峰，并显示数值（约1.08949倍）。
- **关键文字提示：** “即使是最简单的形状，我们也需要无穷多个谐波才能完美描述。逼近永远在进行中。”

### 第六章.3 希尔伯特空间投影（Chapter 3）

- **必须遵守的约束：**
  - 整个场景上方必须有一个永久性标签：**“三维类比：函数即向量”**。
  - 矢量端点拖拽必须约束在单位球面上（满足归一化条件）。使用球坐标约束算法。
  - 第4、5维轴以“折叠”形式出现，不可展开成清晰的欧几里得轴，强化类比边界。
- **数学映射：**
  - 三个基向量对应 `sin(πx/L)`, `sin(2πx/L)`, `sin(3πx/L)`，内积定义为积分 `(2/L)∫sin(mπx/L)sin(nπx/L)dx`。
  - 向量的三个分量 `(c1, c2, c3)` 对应傅里叶系数。
- **正交性演示：** 用户选择两个基，场景动态显示两个正弦波的乘积曲线，并计算出定积分=0的过程动画。

### 第六章.4 量子公理剧场（Chapter 4）

- 这是核心模拟器的阶段3。所有之前解锁的功能在此整合。
- **制备态：** 在希尔伯特空间投影中拖拽向量，势阱中的波函数曲线和概率密度云实时更新。
  - 波函数：`ψ(x) = Σ c_n ψ_n(x)`，使用复数？对于无限深势阱，本征函数可以取实数值。但若展示时间演化，必须包含 `exp(-iE_nt/ħ)` 相位因子，产生干涉。制备态时可设置为 `t=0`。
- **测量：**
  - 用户可选择测量基（能量本征基、位置、动量等）。
  - 以能量基为例：根据 `|c_n|^2` 加权随机选择坍缩结果。执行坍缩后，向量动画跳跃到对应的坐标轴上，波函数瞬间变为纯 `ψ_n(x)`。
  - **重要：** 测量操作必须是一次性的、概率性的，不可逆（除非用户重置）。
- **动量空间切换：**
  - 显示位置波函数的傅里叶变换（数值FFT计算）。
  - 提供“压缩波包”交互：用户在位置空间压缩波函数，动量空间自动展宽，定量显示位置和动量的标准偏差乘积接近 `ħ/2`。

### 第六章.5 思想关系网络（Chapter 5）

- 使用3D力导向图（Force-Directed Graph）呈现知识节点。
- **节点：** 毕达哥拉斯、牛顿、贝克莱、傅里叶、黎曼、康托尔、罗素、希尔伯特、海森堡、薛定谔、冯·诺依曼、哥德尔等。
- **连线类型（必须用不同颜色编码）：**
  - 启发（Inspired by）：蓝色
  - 解决了（Solved）：绿色
  - 挑战（Challenged）：红色
  - 独立发现（Independently discovered）：橙色
- **关键规则：** 哥德尔与海森堡之间**没有直接连线**。点击它们之间的空白才出现关于“理性边界”的反思文字，防止误导性类比。

### 第六章.6 结语（Chapter 6）

- 回到阶段3的模拟器，但界面出现不可点击的灰暗按钮：“同时知道确切位置和动量”。
- 文字以缓入方式浮现，只提问题，不给答案。

## 七、性能与优化约束

- **场景生命周期：** 离开任一卫星空间时，调用 `dispose()` 销毁场景内所有几何体、材质、纹理，并移除事件监听。
- **粒子系统：** 一律使用 `InstancedMesh`，避免单个 `Mesh` 对象的开销。
- **Shader计算：** 波函数实时演化、叠加曲线等，尽可能在GPU中通过自定义Shader计算，避免CPU数组循环。
- **Web Worker：**
  - 傅里叶系数计算、矩阵运算、随机数生成（用于测量坍缩）放入 Worker。
  - 主线程与 Worker 通信使用 `postMessage` + `Transferable` 对象。
- **降级策略：**
  - 检测设备性能（通过 `navigator.hardwareConcurrency`、测试帧率）。
  - 移动端自动进入2.5D模式：场景预渲染为序列帧或使用简单的Canvas动画，交互改为点击热点。

## 八、UI/UX 约束

- **玻璃拟态面板：** `backdrop-filter: blur()`，半透明背景，柔和边框，文字颜色确保对比度。
- **颜色系统：**
  - 深空蓝背景 `#0f172a`
  - 量子蓝 `#3b82f6`
  - 希尔伯特紫 `#8b5cf6`
  - 概率红 `#ef4444`
- **字体：** Inter（UI）、JetBrains Mono（公式、代码片段）。
- **动画：** 所有过渡使用 GSAP 的 `ease`（如 `power2.out`），无生硬跳变。
- **引导性交互：** 当用户首次面对一个可拖拽对象时，鼠标悬停有脉冲提示，移动端有微动效引导触摸。

## 九、开发指令

当请求实现特定功能时，请遵守如下流程：
1. 确认该功能属于核心模拟器还是某个卫星空间。
2. 检查其依赖的认知解锁条件，确保不会跳过前置阶段。
3. 优先保证数学/物理准确性，再进行视觉美化。
4. 所有交互必须提供即时、清晰的视觉反馈。
5. 代码注释中标注任何对精度或抽象概念的简化，并说明原因。

**项目起始命令：** 初始化核心模拟器阶段0，创建基本场景、粒子系统、方势阱盒子模型，并实现经典粒子碰撞。
